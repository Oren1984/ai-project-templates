# AI Project Templates

A personal, evolving repository of **documentation-level templates**  
for Applied AI projects.

This repository defines how I design, document, structure, and close  
real-world AI systems.

---

## Purpose

This repository serves as a **project standards hub**.

It captures:
- How an Applied AI project should be structured
- What artifacts must exist
- How results are validated and documented
- What “Definition of Done” means for AI work

It reflects real executed projects — not theoretical outlines.

---

## What this repository contains

The templates cover Applied AI across:

- **Data Science & Classical ML**
- **Deep Learning & NLP**
- **Large Language Models**
- **RAG Systems & Agents**
- **LLM-powered Applications**

Each template represents:
- A structured execution flow
- Required artifacts (models, metrics, reports)
- Documentation expectations
- Reproducibility standards

This is a **documentation-first architecture approach**.

---

## Core Principles

All templates follow:

- **Documentation-first**
- **Stage-based execution**
- **Artifacts over assumptions**
- **Reproducibility**
- **Clear separation of concerns**
- **Explicit Definition of Done**

AI projects are treated as engineered systems — not experiments.

---

## How to use

1. Choose the relevant domain folder.
2. Copy the appropriate template into a new repository.
3. Adapt placeholders and project-specific details.
4. Ensure all required artifacts are produced before closing the project.

Each template links to a real reference implementation.

---

## What this repository is NOT

- Not a code framework
- Not a scaffolding tool
- Not a tutorial
- Not an academic syllabus copy

It represents a **personal applied AI workflow standard**.
